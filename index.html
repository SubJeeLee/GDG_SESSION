💡 웹 접근성(Web Accessibility)이란?

신체적 불편함(시각, 청각, 운동)이나 환경적 제약(마우스 고장, 햇빛 등)에 상관없이 누구나 동등하게 정보를 이용할 수 있어야 한다는 개념입니다.

시맨틱 태그 (Semantic Tags)

시맨틱 태그(header, nav, main 등)는 페이지의 정보를 명시적으로 설계하여 검색 엔진 로봇이 정보 구조를 쉽게 파악하도록 돕습니다.

Strong 태그의 힘

<strong> 태그를 사용하면 단순 볼드체가 아니라 의미론적 강조가 부여됩니다.

  • 🔊 스크린 리더: 해당 부분을 더 강한 어조로 읽어줌
  • 🤖 검색 엔진: 페이지 내 '중요 문장'으로 인식하여 가점 부여
<!-- Bad Structure -->
<div class="header">...</div>

<!-- Good Semantic -->
<header>
  <nav>...</nav>
</header>
<main>
  <p>이것은 <strong>정말 중요한</strong> 내용입니다.</p>
</main>

🖼️ 이미지 대체 텍스트 (Alt)

alt 태그는 스크린 리더에게 이미지를 설명해주고, 검색 엔진 크롤러에게는 이미지 검색 순위를 높여주는 역할을 합니다.

📝 좋은 Alt 텍스트 예시:

<img src="chick.jpg" alt="알에서 부화한 노란 병아리">

💡 Tip: 단순히 "병아리"라고 적는 것보다, 시각화할 수 있도록 "관련성이 높고 정확하게" 작성하는 것이 중요합니다.

Practical Solutions

접근성을 높이는 3가지 핵심 기둥

단순한 태그 교체를 넘어, 사용자의 경험을 설계하는 구체적인 방법입니다.

견고한 문서 구조 (Structure)

시맨틱 태그와 버튼 Basic

div는 아무 의미가 없습니다. button은 클릭, 키보드 포커스, 엔터/스페이스 키 작동을 모두 무료로 제공합니다.

<!-- Bad: 키보드 접근 불가 -->
<div onClick="login()">로그인</div>
<!-- Good: 모든 기능 내장 -->
<button onClick="login()">로그인</button>

헤딩(Heading) 계층 구조 Important

스크린 리더 사용자의 70%는 h1~h6를 이용해 목차처럼 탐색합니다. 디자인 크기가 아니라 논리적 순서가 중요합니다.

H1: 페이지 제목
H2: 주요 섹션
H3: 하위 콘텐츠

명확한 정보 전달 (Information)

대체 텍스트의 맥락

모든 이미지에 설명이 필요한 건 아닙니다.

  • ✅
    정보 전달: alt="검은색 가죽 자켓"
  • ✅
    장식용: alt="" (빈 값으로 스킵)

친절한 에러 메시지

"빨간 테두리"만으로는 색약 사용자가 알 수 없습니다. 텍스트 설명과 함께 aria-describedby로 연결하세요.

⚠️ 이메일 형식이 올바르지 않습니다.

🌍 올바른 언어 설정 (Lang)

<html lang="ko"> 한 줄이 없으면 스크린 리더는 한국어를 영어 억양으로 읽습니다.

제약 없는 상호작용 (Interaction)

초점(Focus) 가두기

모달이 열렸을 때 키보드 포커스가 뒷배경으로 빠져나가면 사용자는 길을 잃습니다.

보이지 않는 이름표 (ARIA) Pro Tip

아이콘 버튼(검색, 닫기)은 이름이 없습니다. aria-label로 이름을 지어주세요.

"버튼"

vs

"검색하기 버튼"




Developer's Guide

코드로 보는 접근성 패턴

"복잡한 로직이 아닙니다. 올바른 속성 하나가 사용성을 결정합니다."

Structure.tsx
HTML5

// ❌ Bad: 키보드/스크린리더 무시

<div onClick="...">
  로그인
</div>

// ✅ Good: 모든 상호작용 지원

<button
  type="button"
  onClick="..."
>
  로그인
</button>

* Tab 키 포커스, Enter/Space 실행 자동 지원

FocusTrap.js
Logic

// 🔒 포커스 가두기 (Keydown Event) 

if (e.key === 'Tab') {
  if (e.shiftKey) {
    // Shift + Tab (뒤로 가기)
    if (activeEl === firstItem) {
      e.preventDefault();
      lastItem.focus(); // 끝으로 점프
    }
  } else {
    // Tab (앞으로 가기)
    if (activeEl === lastItem) {
      e.preventDefault();
      firstItem.focus(); // 처음으로 점프
    }
  }
}

* 모달 안에서 포커스가 나가지 않고 순환합니다.

InputError.tsx
WAI-ARIA

// ✅ Good: 에러 메시지 연결

<input
  type="email"
  aria-invalid="true"
  aria-describedby="err1"
/>

<span
  id="err1"
  role="alert"
  class="text-red-500"
>
  이메일 형식이 아닙니다.
</span>

* 에러 발생 시 스크린 리더가 즉시 읽어줍니다.

IconButton.tsx
Labeling

// ❌ Bad: 읽을 텍스트 없음

<button>
  <Icon name="close" />
</button>

// ✅ Good: 보이지 않는 라벨 제공

<button
  aria-label="팝업 닫기"
>
  <Icon name="close" />
</button>

* 화면엔 아이콘만, 소리로는 "팝업 닫기" 출력
